2. Процесс тестирования (делай по шагам)

Рекомендуемый порядок прохождения. Каждый шаг — отдельный этап с фиксацией результата.

1
Подготовка
Открой DevTools браузера (F12) → вкладка Console. Включи «Preserve log». Открой в отдельной вкладке эту страницу. Закрой другие вкладки с проектом, чтобы не путать логи. Заведи отдельный текстовый файл qa-report-<твоё-имя>-YYYY-MM-DD.md для заметок в свободной форме.
2
Smoke-тест (10 мин)
Пройди чек-лист «Smoke» ниже — это критический путь пользователя. Если что-то из smoke падает — блокирующий дефект, остальное можно не тестировать пока не починим.
3
Полный обход по модулям
Пройди вкладки Auth → Tenders → Subscriptions → Dashboard → API → UI/UX → Security → Performance. На каждый тест-кейс кликай одну из кнопок: ✅ Pass / ❌ Fail / ⏭ Skip / 🚫 Block. Прогресс сохраняется в localStorage браузера — можно прерваться и продолжить завтра.
4
Сбор дефектов
Все упавшие кейсы → форма «Отчёт о дефекте» внизу страницы. Заполни по одному отчёту на каждый баг. Когда закончишь — нажми «📥 Скачать отчёт JSON», файл сохранится к тебе на диск.
5
Отправка результатов
Отправь JSON-отчёт + свой свободный md-файл Сергею в Telegram (@Pakshin) с пометкой «QA TenderSearch YYYY-MM-DD». Если нашёл блокирующий баг (P0) — сообщи сразу в чат, не дожидаясь конца прогона.

4. Тест-кейсы

Проходи по вкладкам. Кликай Pass/Fail/Skip/Block на каждом кейсе. Результаты сохраняются автоматически в localStorage браузера.

5. Отчёт о дефекте

Заполняй по одному отчёту на каждый баг. Серьёзность (Severity): P0 Blocker — приложение не работает / ломает авторизацию; P1 Critical — основной сценарий сломан; P2 Major — серьёзная проблема, есть workaround; P3 Minor — косметика / UX.
