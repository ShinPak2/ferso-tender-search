## Цель исследования

Выбрать первичный и резервный источники данных о тендерах госзакупок (44-ФЗ, 223-ФЗ, ПП-615) для парсера TenderSearch. Критерии выбора:

1. **Покрытие:** 44-ФЗ + 223-ФЗ + желательно ПП-615
2. **Полнота данных:** наличие ТЗ, проектов контрактов, документов
3. **Стоимость:** вписаться в бюджет проекта ($500 MVP, $300 фаза 2)
4. **Юридическая чистота:** данные должны быть из открытых источников или по лицензии
5. **Техническая реализуемость:** наличие API/FTP/парсинга без captcha-сложностей
6. **Задержка обновления:** не более 24 часов с момента публикации

| Параметр | Значение |
|---|---|
| **Покрытие** | ✅ 44-ФЗ, 223-ФЗ, ПП-615 — полное |
| **Полнота** | ✅ Все поля извещения + ссылки на документы |
| **Стоимость** | ✅ Бесплатно |
| **Юридическая чистота** | ✅ Официальный источник |
| **Документы** | ✅ Все типы (DOC, DOCX, PDF, XLS, XLSX, ZIP) |
| **Задержка** | ⚠️ HTML — до 1 часа после публикации; FTP — раз в сутки |
| **API** | ❌ Официального API нет. Есть схемы XML-выгрузок |
| **Защита от парсинга** | ⚠️ Средняя: User-Agent фильтр, возможны rate-limits, иногда captcha на поиск |

**Плюсы:**
- Официальный источник — данные 100% достоверные
- Бесплатно
- Полное покрытие всех типов закупок
- FTP-выгрузки структурированы (XML по схемам ФЗ-44) — самый надёжный способ

**Минусы:**
- Парсинг HTML хрупкий — меняется вёрстка
- FTP-выгрузки только по 44-ФЗ, 223-ФЗ нужно парсить через поиск
- Защита от ботов — нужно ротировать User-Agent, использовать задержки, решать captcha при массовом парсинге
- Ответственность по 152-ФЗ: собирать только публичные данные (нарушений нет, т.к. данные публичны)

**Вывод:** Первичный источник. Используем FTP для 44-ФЗ (ежедневно в 03:00) + HTML-парсинг поиска для свежих записей (каждый час). Документы скачиваем из карточки закупки.

## Решение для MVP (фаза 1)

| Источник | Роль | Технология | Периодичность |
|---|---|---|---|
| **zakupki.gov.ru FTP** | Основной парсер 44-ФЗ (XML) | Python + lxml + aioftp | Раз в сутки в 03:00 |

| **DaMIA API** | Резерв (если zakupki заблокирует) | httpx + JSON | Подключить при >1000 user'ов |

**Оценка трудозатрат на реализацию MVP-парсера:**
- FTP-парсер XML 44-ФЗ: 1-2 дня (Разработчик)

- Скачивание документов + document_parser: 2 дня (Разработчик)
- Всего: 4-5 дней (вписывается в фазу 2 из PLAN.md)

## Юридические аспекты

По 152-ФЗ «О персональных данных» — все данные в ЕИС публичны (открытые реестры). Парсинг публичных данных zakupki.gov.ru **не нарушает** закон, так как данные находятся в открытом доступе для неограниченного круга лиц (ст. 8 152-ФЗ, Решение ВС РФ от 14.07.2015 по делу № АКПИ15-654).

**Меры предосторожности:**
- Соблюдать `robots.txt` (не игнорировать явно запрещённые разделы)
- Уважительные интервалы (не чаще 1 запроса в 3-5 секунд)
- Указывать контактный email в User-Agent (`TenderSearch/0.1 contact@ferso.ru`)
- Не хранить данные дольше, чем требуется для сервиса
- Не использовать для перепродажи/спама

## Метрики качества источника

Для ежедневного мониторинга (метрики в дашборд Админа):

- `tenders_parsed_today` — количество спарсенных тендеров за день
- `documents_downloaded_today` — количество скачанных документов
- `parser_errors_today` — ошибки парсинга (должно быть <5%)
- `ai_extraction_success_rate` — % успешно извлечённых данных через AI
- `last_ftp_sync` — timestamp последней синхронизации по FTP
